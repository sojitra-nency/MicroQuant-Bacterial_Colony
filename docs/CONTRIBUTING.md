# Contributing to MicroQuant

First off, thank you for considering contributing to MicroQuant! It's people like you that make MicroQuant such a great tool.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* If not, open a new issue. Clearly describe the problem and include steps to reproduce the bug.

### Suggesting Enhancements
* Open a new issue with a clear title and description of the suggested enhancement.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the project's coding standards (PEP 8, docstrings).

## Coding Standards

* **Style**: Follow PEP 8.
* **Documentation**: Use Google-style docstrings for all modules, classes, and functions.
* **Type Hints**: Use Python type hints for better code clarity and IDE support.
